# Universal GraphRAG Scratchpad Middleware

**Version:** 3.0.0 — Universal Multi-Backend Architecture
**Target Environment:** Localhost / Cloud (CPU-bound Middleware + VRAM-bound LLM)
**Core Paradigms:** Database-Backed Hierarchical Graph Views · Universal Inference Routing · Louvain Community Compression · Token Knapsack Optimization · Deterministic Hallucination Guardrails

---

## 🧠 What Is This?

A **universal middleware layer** that sits between any downstream LLM and a stream of messy, unstructured input. It ingests chaotic raw text (logs, documents, agent outputs), extracts structured knowledge graph triplets, compresses dense subgraphs automatically, and returns a clean, token-bounded Markdown context window ready for any LLM to consume.

Works as a **Python SDK**, a **FastAPI REST microservice**, or both simultaneously.

## 2. What's New in v3.0 (This Update)

### ✅ Universal Inference Layer (`src/inference.py`)
A new `UniversalInferenceEngine` class routes every LLM call through a clean strategy interface. Configure the backend once via environment variables — the engine handles all grammar enforcement automatically.

| Backend | Enforcement Mechanism | Model Examples |
|---|---|---|
| **Ollama** | Native JSON Schema → GBNF grammar | `qwen2.5:14b`, `gemma2:9b` |
| **Llama.cpp** | HTTP `json_schema` parameter (token masking) | Any GGUF model via server |
| **Transformers** | `instructor` library logit processors | Mistral-Nemo-12B, AWQ/GPTQ |

### ✅ Hierarchical Knowledge Graph (L1 → L2 Compression)
The `knowledge_graph` table now supports two hierarchy levels:
- **L1 (Raw Facts):** Extracted directly from raw text by agents.
- **L2 (Compressed Summaries):** Auto-generated by the background sweeper when node density crosses a threshold.

Compressed L1 nodes are marked `is_active = FALSE` and linked via `parent_node_id` to their L2 parent — enabling full drill-down retrieval.

### ✅ Entity Disambiguation Engine (`src/engine.py`)
Before any entity is written to the graph, it is canonicalized via `rapidfuzz` fuzzy matching against all existing active nodes. This prevents synonym fragmentation (e.g., `FASTAPI_ROUTER` vs. `FAST_API_ROUTER` collapsing into a single canonical node).

### ✅ Background Graph Sweeper Daemon (`src/sweeper.py`)
Refactored into a `GraphSweeperDaemon` class:
- Runs as an `asyncio.Lock`-protected background task, protecting the FastAPI event loop.
- Partitions the graph **per session** before clustering.
- Uses Louvain detection with a fixed `random_state=42` for deterministic results.
- Calls the `UniversalInferenceEngine` to generate real `L2CompressionPayload` objects.
- Performs the L1 → L2 database swap inside a single atomic transaction.

### ✅ Universal Python SDK (`src/client.py`)
```python
from client import ScratchpadMiddleware

memory = ScratchpadMiddleware()
session_id = "my_agent_session"

# 1. Feed messy input (logs, documents, agent outputs...)
context = memory.process_turn(session_id, raw_messy_text)

# 2. Feed context directly to ANY LLM
response = openai.ChatCompletion.create(
    model="gpt-4o",
    messages=[{"role": "system", "content": context}, {"role": "user", "content": "What's the security risk?"}]
)
```

### ✅ New REST Endpoints (`src/main.py`)
Two new endpoints exposed for any non-Python client (Node.js, Go, curl, etc.):

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/v1/middleware/process` | Ingest messy input → return clean scratchpad |
| `POST` | `/v1/middleware/drill-down` | Expand a compressed L2 node back to its L1 history |

## 3. The Data Schema & Memory Model

The architecture abandons flat file persistence in favour of a hierarchical, atomic SQLite schema.

| Table | Purpose |
|---|---|
| `sessions` | Tracks session status and Master Plan |
| `knowledge_graph` | Core L1/L2 triplet store with lineage tracking |
| `unresolved_variables` | Active "to-do" context gaps for the agent swarm |

**Key `knowledge_graph` columns:**

```sql
edge_id          TEXT PRIMARY KEY
session_id       TEXT NOT NULL
source_entity    TEXT NOT NULL      -- Canonicalized UPPERCASE
relationship     TEXT NOT NULL      -- lowercase_with_underscores
target_entity    TEXT NOT NULL      -- Canonicalized UPPERCASE
citation_quote   TEXT               -- Exact substring from raw text (L1 only)
hierarchy_level  INTEGER DEFAULT 1  -- 1=raw, 2=compressed
parent_node_id   TEXT DEFAULT NULL  -- Points to L2 parent edge_id
relevance_score  REAL DEFAULT 1.0   -- Centrality/decay score
is_active        BOOLEAN DEFAULT 1  -- FALSE when compressed
```

## 4. End-to-End Execution Pipeline (Sync-Act-Commit)

### Phase 1: SYNC (Context Retrieval)
1. The Agent SDK calls `GET /v1/session/{session_id}/memory`.
2. The View Generator queries SQLite for all `is_active = TRUE` triplets, ordered by `relevance_score DESC`.
3. L2 compressed nodes are tagged with `[COMPRESSED | ID: <edge_id>]` and a drill-down affordance.
4. The Markdown scratchpad is returned to the agent.

### Phase 2: ACT (Local Inference)
1. The agent appends the Markdown view to its system prompt.
2. The agent calls its local LLM via the `UniversalInferenceEngine` — Ollama, Llama.cpp, or Transformers.
3. The LLM outputs strict JSON matching the `PageExtractionPayload` Pydantic schema (grammar-enforced).

### Phase 3: COMMIT (Verification & Storage)
1. The Agent SDK sends extracted triplets + raw text to `POST /v1/agent/update`.
2. The `canonicalize_entity` disambiguation gate collapses synonym entities.
3. The citation verification gate checks each `citation_quote` exists as an exact substring.
4. Verified triplets are written atomically to SQLite. Hallucinated ones are rejected.
5. If all triplets are rejected, an HTTP 422 is returned and the agent retries.

## 5. Background Graph Sweeper

The `GraphSweeperDaemon` runs as a background coroutine alongside FastAPI:

```python
# Triggered inside @app.on_event("startup")
asyncio.create_task(graph_maintenance_daemon())
```

**Sweep Execution Steps:**
1. **Trigger Evaluation:** SQL `GROUP BY` aggregation counts node degrees per session.
2. **Louvain Clustering:** Load active edges into `networkx.Graph()`, run `best_partition(G, random_state=42)`.
3. **SLM Condensation:** Filter communities above `min_community_size`. For each, call the `UniversalInferenceEngine` with the community subgraph and receive a validated `L2CompressionPayload`.
4. **Atomic Swap:** Insert L2 summary nodes, then `UPDATE ... SET is_active = FALSE, parent_node_id = ?` for all collapsed L1 edges in a single transaction.

## 10. Failure Handling & Resilience

| Failure Mode | Mitigation |
|---|---|
| Concurrent agent writes | SQLite WAL mode queues transactions atomically |
| SLM schema hallucination | Pydantic + grammar enforcement blocks malformed output |
| Synonym entity drift | `rapidfuzz` fuzzy canonical matching before every insert |
| Mid-execution crash | All state in SQLite; agents resume from last committed triplet |
| Overlapping sweeper runs | `asyncio.Lock()` prevents concurrent Louvain executions |
| LLM backend failure | `UniversalInferenceEngine` raises typed exceptions for clean retry |
